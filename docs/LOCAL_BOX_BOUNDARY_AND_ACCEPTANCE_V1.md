# LOCAL_BOX_BOUNDARY_AND_ACCEPTANCE_V1

## 1. 结论

- `local_box` 当前只允许先定义边界与单一验收标准。
- 在本文件获批并落库前，不进入 `local_box/` 代码或配置修改。

## 2. 本轮边界

- 允许：
  - 仅在 `docs/` 新增本文件
  - 只写边界、范围、验收标准、回滚口径
- 禁止：
  - 修改 `local_box/` 下任何代码或配置
  - 修改 `execution_service/`
  - 修改 `risk_engine/`
  - 修改 `shared/`
  - 修改 `cloud/`
  - 修改 `anchor-backend/`
  - 修改子模块
  - 并行新增第二个 active 目标

## 3. local_box 当前只讨论什么

- 只讨论：
  - `local_box` 的最小职责边界
  - 与现有主线的边界关系
  - 下一步唯一允许推进的最小验收对象

- 不讨论：
  - 更大架构重构
  - 多模块一起联动改造
  - 新平台接入
  - 未定义验收的功能扩张

## 4. local_box 最小边界

- `local_box` 只应被定义为一个本地最小执行边界 / 控制边界候选。
- 当前阶段不把它扩写为完整新主线，不替代现有已封板主线。
- 当前阶段只允许先明确：
  - 它最小负责什么
  - 它明确不负责什么
  - 如何用一个单一验收判断它是否值得进入下一步

## 5. 唯一下一验收目标

- 下一步唯一允许推进的对象：
  - **只定义 1 个 `local_box` 最小验收目标**
- 该目标必须同时满足：
  - 单文件或单路径最小改动
  - 可直接验证 PASS / FAIL
  - 可直接回滚
  - 不依赖并行改多个目录

## 6. 单一验收标准模板

- 必须明确写出：
  - 目标文件或目标路径
  - 允许修改范围
  - 禁止修改范围
  - PASS 条件
  - FAIL 条件
  - 回滚方法

## 7. 当前正式约束

- 在新的唯一验收目标写死并获批前：
  - 不改 `local_box/`
  - 不改相关执行链代码
  - 不做“顺手补齐”
  - 不从文档直接跳到实现

## 8. 回滚口径

- 本轮若仅新增本文档：
  - 删除 `docs/LOCAL_BOX_BOUNDARY_AND_ACCEPTANCE_V1.md`
  - 不涉及代码回滚

## 9. 一句话判断

- `local_box` 当前还不是进入实现阶段的主线对象；现在只允许先把边界和单一验收标准写死。
