# PROJECT_ANCHOR_READONLY_DEMO_ENTRY_PLAN_V1

本文档与 `PROJECT_ANCHOR_EARLY_DEMO_SCOPE_V1.md` 配套：在 Early Demo 边界已写死的前提下，定义**只读演示入口**的设计方案。  
本轮只做方案与验收定义，不实现页面、不改 backend/worker、不改部署。

---

## 1. 展示给谁看

| 受众 | 目的 | 约束 |
|------|------|------|
| 内部产品/工程 | 验证流程与运维叙事是否清晰 | 可访问 staging 或受控 VPN 后的只读入口 |
| 可信外部评审（投资人、合作方、资深用户） | 理解纪律与风控能力边界 | 仅只读；需事先签署或口头确认「演示非交易」 |
| 公众/匿名访客 | 仅允许「一页说明 + 免责声明」级别 | 不开放任何真实 API 或内网路由直连 |

**明确不做：** 面向匿名用户的完整控制台、注册即用的「试用交易」、任何可触发真实资金的入口。

---

## 2. 展示哪几个页面（建议优先级）

以下均为**只读**视角；具体路由以现有或后续只读包装为准，名称表示能力面而非必须同一 URL。

| 优先级 | 页面/能力面 | 展示内容 |
|--------|-------------|----------|
| P0 | 入口说明页 | 一句话定位、Early Demo 边界、免责声明链接（对齐 Early Demo Scope §5） |
| P1 | 运维健康摘要 | `cloud_runtime_check_readonly.sh` 的 PASS/FAIL 摘要（可复制粘贴或截图级，不暴露内网细节） |
| P2 | `/ops` 状态摘要 | kill_switch、worker 心跳、近期事件等**脱敏后的**只读摘要 |
| P3 | `/commands` 命令流 | 仅展示**模拟或历史演示**命令流；不展示真实账户或真实 API 标识 |
| P4 | `/checklist` / `/recovery` | 流程与恢复边界说明；只读，不可改 checklist 状态（若当前为可写，入口层必须禁用写操作） |

**P0 为对外最低集**；P1–P4 可按受众逐步开放，且须在入口页标明「演示环境 / 非交易」。

---

## 3. 哪些话不能说（对外文案红线）

与 Early Demo Scope §4 一致，入口与所有子页必须遵守：

- 不说「自动帮你赚钱」「托管交易」「已接入真实交易所下单」
- 不说「保证收益」「无风险」「替代专业投顾」
- 不说「已生产就绪可接用户资金」（除非单独 go-live 文档明确且与本 Early Demo 分离）
- 不暗示「填写 API key 即可真实下单」作为演示路径的一部分

**必须常驻的短句（可放在页脚）：**  
「当前为 Early Demo，仅展示流程与风控能力，不执行真实交易，不接资金，不接真实 API key。」

---

## 4. 哪些按钮不能开放

| 类型 | 要求 |
|------|------|
| 真实下单 / 真实撤单 / 绑定交易所 | **禁止**出现在演示入口 |
| 提交真实 API key / 保存生产 secret | **禁止** |
| 修改生产 env、compose、nginx、UFW、sudoers | **禁止** |
| 任意「执行 shell」「远程写文件」类控制台 | **禁止** |
| 可写 checklist / 可改 kill_switch / 可触发 worker 写路径 | **禁止**在对外只读演示中开放；内部评审若需，须单独入口与权限 |
| 「一键部署」「compose up/down」 | **禁止**在对外只读演示中开放 |

**允许：** 刷新只读数据、展开/折叠说明、复制 PASS/FAIL 文本、跳转至公开文档（如本仓库内已公开的 scope 文档的静态副本）。

## 5. 验收怎么判定

### 5.1 方案级验收（本轮文档）

- [ ] 受众分层与 P0–P4 页面清单已写清
- [ ] 文案红线与页脚必选句已写清
- [ ] 禁用按钮/能力表已写清
- [ ] 与 `PROJECT_ANCHOR_EARLY_DEMO_SCOPE_V1.md` 无冲突

### 5.2 实现后验收（未来轮次，供引用）

| 项 | 合格标准 |
|----|----------|
| 入口 | 首次加载即见免责声明与 Early Demo 边界 |
| 只读 | 浏览器网络面板中，演示路径无 POST/PUT/PATCH/DELETE 到敏感写接口（或后端对演示身份统一 403） |
| 无密钥 | 页面与 URL 不出现真实 API key、连接串中的密码 |
| 运维摘要 | 展示的 health / 检查结果与 `cloud_runtime_check_readonly.sh` 或等价只读源一致 |
| 回滚 | 关闭演示入口或撤销路由后，生产行为与未开演示前一致 |

**本轮结论：** 仅完成 §5.1；§5.2 在「只读演示入口实现」轮次执行。
